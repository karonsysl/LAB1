# Задание 1. Генерация списков

## Задача 1

### Текст задачи

Создать список из n одинаковых элементов (элемент вводится)

### Алгоритм решения

Предлагаем пользователю  ввести значение элемента списка(element).

Сохраняем введённую пользователем строку в переменную.

Начинается блок обработки ошибок:

Мы превращаем его в число. Проверяем: корректный ли формат ввода.

Если всё хорошо - возвращаем число. Иначе выдает сообщение об ошибке и рекурсивный вызов этой же функции для повторного ввода.

Аналогично предлагаем пользователю ввести количество элементов списка(size), проверяем корректность: не отрицательное ли оно, корректный ли формат ввода.

Далее идет вызов рекурсивной функции построения списка createList. Функция принимает два аргумента- элемент списка element, размер списка size и создаёт список длиной size, все элементы которого равны element.

Затем производится вывод списка на экран пользователя.

# Задание 2. Рекурсия

## Задача 1

### Текст задачи

Найти произведение цифр натурального числа

### Алгоритм решения

Предлагаем пользователю ввести значение element.Сохраняем введённую строку.

Пробуем преобразовать строку в число (int element):

Если число отрицательное - сообщение об ошибке, повторный ввод.

Если строка не число - сообщение об ошибке, повторный ввод.

Иначе сохраняем число.



Вызываем рекурсивную функцию productDigits:

Если число однозначное - возвращаем число.

Иначе:

Берём последнюю цифру (digit = n % 10).

Остаток числа без последней цифры (rest = n / 10).

Рекурсивно перемножаем цифру на произведение оставшихся цифр.

Если введён 0 - результат = 0.



Выводим результат на экран.

# Задание 3.

## Задача 1

### Текст задачи

 Создайте собственные функции для выполнения основных операций над списками (добавление/

удаление/поиск элемента, сцепка двух списков, получение элемента по номеру).

### Алгоритм решения

Пользователю предоставляется меню с возможными вариантами выполнения основных операций над списками

Пользователь выбирает необходимое действие

Если необходимо Создать список:

&nbsp;	При вводе пользователем элементов и количества элементов списка срабатывает проверка на корректность данных с помощью рекурсивных функций readSize и readElement 

Если необходимо Добавить элемент в конец списка :

&nbsp;	Вызов рекурсивной функции addElement

&nbsp;	Если список пустой : возвращаем список с одним элементом.

&nbsp;	Если список не пустой:

&nbsp;	Берём первый элемент.

&nbsp;	Рекурсивно обрабатываем оставшуюся часть.

&nbsp;	Собираем список обратно.

Если необходимо Удалить элемент:

&nbsp;	Вызов рек функции

&nbsp;	Если список пуст- возвращаем пустой список.

&nbsp;	Если список не пуст:

&nbsp;	Проверяем первый элемент.

&nbsp;	Если он равен удаляемому-пропускаем его (возвращаем хвост).

&nbsp;	Если не равен-сохраняем его и продолжаем поиск в хвосте.

&nbsp;	Рекурсия обрабатывает список до конца.

&nbsp;	Результат - список без первого найденного вхождения элемента.

Если необходимо Получить элемент по индексу:

 	Вызов рек функции getByIndex

&nbsp;	Если список пуст - ошибка (индекс вне диапазона).

&nbsp;	Если индекс = 0 - возвращаем первый элемент.

&nbsp;	Если индекс > 0

&nbsp;	пропускаем голову,

&nbsp;	уменьшаем индекс,

&nbsp;	ищем в хвосте.

&nbsp;	Если индекс отрицательный -ошибка.

&nbsp;	Рекурсия продолжается, пока:не найдём элемент,или не закончится список.

Если необходимо найти элемент: 

 	Вызов рек функции contains

&nbsp;	Если список пуст - false.

&nbsp;	Если голова списка равна элементу-true.

&nbsp;	Если не равна- ищем дальше в хвосте.

&nbsp;	Процесс идёт рекурсивно, пока не найдём элемент, или не дойдём до конца списка.

&nbsp;	Далее выводим сообщение о наличии или отсутствии элем

Если необходимо Сцепить со вторым списком:

 	Запрашиваем ввод элементов и количества элементов для второго списка с проверкой корректности ввода

 	Вызов рек функции concat

 	Если первый список пуст - возвращаем второй.

 	Если нет:

 	берём первый элемент,

 	объединяем хвост с вторым списком,

 	собираем список обратно.

 	Процесс идёт рекурсивно, пока не обработаем все элементы первого списка.



При необходимости печатаем список
